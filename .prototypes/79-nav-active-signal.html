#79 · nav active-state signal: A monochrome vs B amber

These indicators mean navigation, not live status, so they lose green. Drag the scroll slider to sweep the minimap through its real entering, active, and passed ramps. Click FAQ rows and minimap nodes to move the active state. Both columns share the same state.

A · Monochrome

Active = bright filled paper. Done, current, upcoming differ by fill and size only. No signal color anywhere on the rail or the FAQ list.

A spec
- minimap active dot: fill var(--paper), glow 0 0 12px rgba(paper, .5), scale 1.35
- minimap passed dot: fill var(--paper-soft), scale 1, no glow
- minimap upcoming dot: outline only (unchanged)
- tick + fill + bleed: paper tones, no green
- faq open LED: fill var(--paper), glow 0 0 8px rgba(paper, .45)

B · Amber

The active node borrows brand amber. Done and upcoming stay monochrome. Known risk: dilutes the CTA meaning of amber.

B spec
- minimap active dot: fill var(--amber), glow 0 0 12px var(--amber-glow), scale 1.35
- minimap passed dot: fill var(--paper-soft), scale 1, no glow
- minimap upcoming dot: outline only (unchanged)
- tick + bleed: amber; fill stays paper-soft
- faq open LED: fill var(--amber), glow 0 0 9px var(--amber-glow)
